Setup & Usage Guide

Complete guide to installing, configuring, and running the Twin2Clouds cost optimization platform.

Prerequisites

  • Docker (recommended) or Python 3.9+
  • Cloud Provider Credentials:
    • AWS: Access Key ID, Secret Access Key, Region
    • Azure: Subscription ID, Tenant ID, Region
    • GCP: Service Account JSON, Project ID, Region
  • Internet Connection: For fetching pricing data from cloud provider APIs

Installation

Option 1: Docker (Recommended)

# Clone the repository
git clone <repository-url>
cd 2-twin2clouds

# Build and start the Docker container
docker-compose up -d

# The application will be available at http://localhost:5003

Option 2: Local Python Installation

# Clone the repository
git clone <repository-url>
cd 2-twin2clouds

# Install dependencies
pip install -r requirements.txt

# Run the application
python rest_api.py

# The application will be available at http://localhost:5003

Configuration

1. Cloud Provider Credentials

Create or edit config/config_credentials.json:

{
  "aws": {
    "aws_region": "eu-central-1",
    "aws_access_key_id": "YOUR_AWS_ACCESS_KEY",
    "aws_secret_access_key": "YOUR_AWS_SECRET_KEY"
  },
  "azure": {
    "azure_region": "westeurope",
    "azure_subscription_id": "YOUR_SUBSCRIPTION_ID",
    "azure_tenant_id": "YOUR_TENANT_ID"
  },
  "gcp": {
    "gcp_region": "europe-west1",
    "gcp_project_id": "YOUR_PROJECT_ID",
    "gcp_service_account_path": "path/to/service-account.json"
  }
}

2. Service Configuration (Optional)

The application uses default configurations in config/service_calc_params.json. You can customize service parameters if needed.

Usage

Web Interface

  1. Navigate to http://localhost:5003/ui
  2. Select your currency (USD or EUR)
  3. Configure your Digital Twin scenario:
    • Layer 1: Number of devices, sending interval, message size
    • Layer 2: Hot, Cool, and Archive storage durations
    • Layer 4: 3D model requirements and entity count
    • Layer 5: Dashboard users and refresh rates
  4. Click "Calculate Cost"
  5. View the optimized cloud provider selection for each layer

REST API

The application provides a comprehensive REST API. View the interactive API documentation at:

  • Swagger UI: http://localhost:5003/docs
  • ReDoc: http://localhost:5003/redoc

Example API Request

PUT /api/calculate
Content-Type: application/json

{
  "numberOfDevices": 100,
  "deviceSendingIntervalInMinutes": 2,
  "averageSizeOfMessageInKb": 0.25,
  "hotStorageDurationInMonths": 1,
  "coolStorageDurationInMonths": 3,
  "archiveStorageDurationInMonths": 12,
  "needs3DModel": false,
  "entityCount": 0,
  "amountOfActiveEditors": 2,
  "amountOfActiveViewers": 10,
  "dashboardRefreshesPerHour": 4,
  "dashboardActiveHoursPerDay": 8,
  "currency": "USD"
}

Fetching Pricing Data

The application automatically caches pricing data for 7 days. To manually refresh:

GET /api/fetch_up_to_date_pricing

This endpoint:

  • Fetches current pricing from AWS, Azure, and GCP APIs
  • Saves data to pricing/fetched_data/pricing_dynamic.json
  • Uses cached data if less than 7 days old

Caching Behavior

The application implements intelligent caching to minimize API calls:

Data Type Cache Duration File Location
Cloud Pricing 7 days pricing/fetched_data/pricing_dynamic.json
Region Lists 7 days pricing/fetched_data/[provider]_regions.json
Currency Rates 1 day pricing/fetched_data/currency.json

Troubleshooting

Common Issues

  • Pricing fetch fails: Verify cloud provider credentials are correct and have necessary permissions
  • Currency conversion errors: Ensure internet connection is available to fetch exchange rates
  • Port already in use: Change the port in rest_api.py or stop the conflicting service
  • Missing credentials: Create config/config_credentials.json with valid cloud credentials

Debug Mode

Enable debug logging by setting the DEBUG environment variable:

export DEBUG=True
python rest_api.py
