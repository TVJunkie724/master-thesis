Future Work

This page documents planned architectural improvements and enhancement opportunities. These items are not required for the current functionality but would improve code quality, maintainability, and adherence to design patterns.

Strategy Pattern: Full Encapsulation

Current State

Calculator classes (AWSCalculator, AzureCalculator, GCPCalculator) currently delegate to standalone functions: 

class AWSCalculator:
    def calculate_data_acquisition(self, params, pricing):
        return calculate_aws_cost_data_acquisition(...)  # Delegation
Proposed Change

Move the formula logic directly into the Calculator class methods, eliminating the standalone functions.

Pros
  • Single source of truth for provider logic
  • True OOP encapsulation
  • No redundant wrapper layer
  • Cleaner architecture
Cons
  • ~50 functions to move across 3 files
  • Formula tests must be updated
  • Higher refactoring risk
  • Class files become larger
Implementation Steps
  1. Move calculate_aws_* function bodies into AWSCalculator methods
  2. Update test_formulas_aws.py to test class methods
  3. Remove unused standalone functions
  4. Repeat for Azure and GCP

Estimated Effort: 4-6 hours · Risk: Medium
