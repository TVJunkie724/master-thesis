Cloud Credentials Setup

Configure cloud provider credentials for fetching real-time pricing data from cloud APIs.

Overview

The Twin2Clouds Optimizer fetches live pricing data from cloud provider APIs to calculate accurate cost estimates. Each provider requires specific credentials to access their pricing APIs.

Minimal Permissions Unlike the Deployer which needs full infrastructure permissions, the Optimizer only needs read-only access to pricing APIs. This means simpler setup and lower security risk.

Credential Storage

Credentials are stored in config/config_credentials.json:

{
  "aws": {
    "aws_access_key_id": "YOUR_ACCESS_KEY",
    "aws_secret_access_key": "YOUR_SECRET_KEY",
    "aws_region": "eu-central-1"
  },
  "azure": {
    "azure_subscription_id": "...",
    "azure_region": "westeurope"
  },
  "google": {
    "gcp_project_id": "...",
    "gcp_region": "europe-west1"
  }
}
Security Warning Never commit config_credentials.json to version control! This file is in .gitignore.

Verification

After configuring credentials, use the built-in credential checker to verify your setup:

# REST API - Verify AWS credentials
GET /permissions/verify/aws

# REST API - Verify GCP credentials
GET /permissions/verify/gcp

# REST API - Verify Azure configuration
GET /permissions/verify/azure

You can also use cURL to test directly:

curl http://localhost:5003/permissions/verify/aws
curl http://localhost:5003/permissions/verify/gcp
curl http://localhost:5003/permissions/verify/azure
