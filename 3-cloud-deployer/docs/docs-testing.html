Testing Guide

Automated test suite ensuring deployment logic correctness and API reliability using mocked AWS services.

Test Categories

Unit Tests

1. Configuration & Utilities

Purpose: Validates internal utility functions and configuration loading logic.

Location: tests/unit/

  • Global configuration loading
  • Credentials validation
  • File path resolution & zipping utils

Integration Tests (Mocked)

2. AWS Deployment Logic

Purpose: Verifies that deployer functions make the correct API calls to create cloud resources.

Location: tests/integration/aws/

Technology: Uses moto to simulate AWS services (IAM, Lambda, IoT, DynamoDB) without real network calls or costs.

  • L1: IAM Role, IoT Rule, Dispatcher creation
  • L2: Event checkers and persisters
  • Cleanup (Destroy) logic verification

3. REST API & Endpoints

Purpose: Tests the FastAPI router and endpoint logic.

Location: tests/api/

  • Endpoint availability
  • Parameter validation
  • Mocked deployer linking

Running Tests

Manual Execution (Docker)

Tests are designed to run inside the Docker container where all dependencies (including moto) are installed.

# Run all tests (Recommended)
docker exec master-thesis-3cloud-deployer-1 pytest -v tests/

# Run only AWS integration tests
docker exec master-thesis-3cloud-deployer-1 pytest -v tests/integration/aws/

# Run with coverage via pytest-cov (if installed)
# docker exec ... pytest --cov=src tests/
Note: Integration tests use moto which mocks AWS. No real AWS credentials are required, and no costs are incurred.

Test Organization

The project follows a modular testing structure:

tests/
├── conftest.py              # Shared fixtures (Environment mocks, Global config mocks)
├── unit/
│   ├── test_globals.py      # Config loading tests
│   └── test_util.py         # Utility tests
├── integration/
│   ├── aws/
│   │   ├── conftest.py      # AWS-specific fixtures (moto clients)
│   │   └── test_core_deployer_aws.py  # L1-L5 deployment verification
│   ├── azure/               # (Planned) Azure integration tests
│   └── google/              # (Planned) GCP integration tests
└── api/
    └── test_rest_api.py     # FastAPI endpoint verification
