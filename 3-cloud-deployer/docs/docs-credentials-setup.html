Cloud Credentials Setup

Configure cloud provider credentials and permissions required for deploying your Digital Twin infrastructure.

Overview

The Multi-Cloud Digital Twin Deployer requires specific permissions on each cloud provider to create, manage, and destroy infrastructure resources. Each provider has its own authentication mechanism and permission model.

Principle of Least Privilege We provide ready-to-use policy documents containing only the permissions required by the deployer. For development, you can use broader admin policies, but for production we recommend using these minimal policies.

Credential Storage

All credentials are stored in your project's config_credentials.json file:

{
  "aws": {
    "aws_access_key_id": "YOUR_ACCESS_KEY",
    "aws_secret_access_key": "YOUR_SECRET_KEY",
    "aws_region": "eu-central-1"
  },
  "azure": {
    "azure_subscription_id": "...",
    "azure_tenant_id": "...",
    "azure_client_id": "...",
    "azure_client_secret": "...",
    "azure_region": "westeurope"
  },
  "google": { ... }
}
Security Warning Never commit config_credentials.json to version control! This file is in .gitignore.

Verification

After configuring credentials, use the built-in credential checker to verify your setup:

# CLI - Verify AWS credentials
python main.py
>>> check_credentials aws

# CLI - Verify Azure credentials
>>> check_credentials azure

# REST API
GET /credentials/check/aws
GET /credentials/check/azure

The credential checker validates that your credentials have all required permissions for each deployment layer.
