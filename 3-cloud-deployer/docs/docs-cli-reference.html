CLI Reference

Complete reference for the interactive command-line interface.

Starting the CLI

cd src
python main.py

You'll see the interactive prompt:

Welcome to the Digital Twin Manager. Type 'help' for commands.
>>>

Available Commands

deploy

Deploy the entire digital twin environment.

Usage:

>>> deploy

Actions:

  • Runs sanity checks on configuration
  • Deploys core infrastructure (Lambda, storage, TwinMaker, Grafana)
  • Deploys IoT devices and processors
  • Creates entity hierarchy in TwinMaker
  • Deploys event action Lambda functions
  • Initializes device shadow values

Typical Time: 5-15 minutes

destroy

Destroy all deployed resources.

Usage:

>>> destroy

Warning: This deletes all cloud resources and data. The operation is irreversible!

Actions (in reverse dependency order):

  • Deletes initialization values
  • Deletes event actions
  • Deletes entity hierarchy
  • Deletes IoT devices and processors
  • Deletes core infrastructure

info

List all deployed resources and their status.

Usage:

>>> info

Output includes:

  • Lambda functions with ARNs
  • IoT Things and certificates
  • Storage resources (DynamoDB tables, S3 buckets)
  • TwinMaker workspace and entities
  • Grafana workspace

help

Show available commands.

Usage:

>>> help

exit

Exit the CLI.

Usage:

>>> exit

Alternatively, press Ctrl+C or Ctrl+D

Tips & Best Practices

  • Use the REST API for automation: The CLI is great for interactive use, but use the REST API for CI/CD pipelines
  • Check info regularly: Run info to verify deployment status
  • Deploy incrementally: Use the REST API's layer-specific endpoints (/deploy_l1, etc.) for better control
  • Review logs: Check console output for detailed progress and error messages
