CLI Reference

Complete reference for the interactive command-line interface.

Starting the CLI

cd src
python main.py
Interactive Prompt: Welcome to the Digital Twin Manager. Type 'help' for commands.

Deployment & Destruction

Core Commands
Command Description Example
deploy [provider] Deploy entire environment (L1-L5). deploy aws
destroy [provider] Destroy entire environment. Irreversible! destroy aws
Granular Operations

Manage individual layers (L1: IoT, L2: Compute, L3: Storage, L4: TwinMaker, L5: Grafana).

deploy_l1 ... deploy_l5 Deploy specific layer. deploy_l1 aws
destroy_l1 ... destroy_l5 Destroy specific layer. destroy_l1 aws
Events

recreate_updated_events: Redeploy only Event Actions and Checker logic for rapid iteration.

Verification & Status

Command Description
info Show summary of deployed resources (ARNs, IDs).
check Run verification checks for all layers.
check_l1 ... check_l5 Run checks for a specific layer.

Configuration Inspection

Inspect currently loaded configuration files.

  • info_config_hierarchy

Lambda Management

Command Usage Description
lambda_update lambda_update <name> Zip and update code for local function. Use default-processor to update all processors.
lambda_logs lambda_logs <name> [n] Fetch last n logs from CloudWatch.
lambda_invoke lambda_invoke <name> [payload] [sync] Invoke function synchronously or asynchronously.

Other

  • help - Show the help menu.
  • exit - Exit the CLI (or Ctrl+C).

Tips & Best Practices

  • Use the REST API for automation: The CLI is great for interactive use, but use the REST API for CI/CD pipelines
  • Check info regularly: Run info to verify deployment status
  • Deploy incrementally: Use the REST API's layer-specific endpoints (/deploy_l1, etc.) for better control
  • Review logs: Check console output for detailed progress and error messages
